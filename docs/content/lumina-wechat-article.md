## 正文

### 开头

组会群里又有人发人格测试链接了。这次是十六型，上次是九型，再上次是某个测"你是哪种猫"的。点进去，答完六十道模棱两可的题，得到一段放之四海而皆准的描述，配一张精致的分享卡片。你明知道这东西的心理测量学基础站不住，但还是转发了，因为"E 人 I 人"已经成了实验室的社交货币。

我们对这类测试有两个不满。第一，它们的"科学感"是装出来的：没有公开的计分方式，没有稳定性检验，却敢用"荣格理论"给自己背书。第二，它们的题目离实验室太远——"你在聚会上感到精力充沛吗"？我上一次聚会是在等离心机转完的十五分钟里吃了半盒饼干。

所以我们做了 LBTI（Laboratory Behaviour Type Indicator，实验室行为类型指标）。它是一个公开声明自己是虚构作品的人格测试：21 种科研人格全是编的，但从你的答案到最终结果之间的每一步计算——距离、PCA、聚类、熵、自助重采样——都是真实的、确定性的、可以在方法页逐条查验的数学。

换句话说：市面上的测试用假数学包装真忽悠，LBTI 用真数学包装明码标价的虚构。

- 用途：让读者一眼看懂整个系统在做什么。
- 构图建议：横向流程图，仿生信流水线风格——"回答采集 → 完整性检查 → 特征向量构建 → 分数归一化 → PCA 投影 → 原型距离 → 稳定性重采样 → 人格注释"，八个节点用柔和的纸质底色卡片串联，节点下标注一行小字说明。

### 一、它到底是个什么东西

LBTI 是一个纯前端的网页应用：36 道五点量表题，题目全部来自湿实验和计算科研的真实场景——跑胶、流式、荧光显微、SEM、测序、生信分析、ODE 建模。比如第一题不是"你喜欢计划吗"，而是"我更喜欢先提出一个生物学或物理机制，再开始收集或分析数据"。

答完之后，计分引擎把你的答案变成 15 个维度上的分数，分成五组：提问方式、对证据的态度、执行风格、协作模式、可持续性。然后在这个 15 维空间里，用加权欧氏距离找出离你最近的"人格原型"——21 个里的一个。

这些人格的名字大概能让你猜到项目的画风：**先验炼金术士（BAYES）**、**移液枪战士（PIPET）**、**审稿人二号转世（R2D2，标语是"再加一个对照。永远。"）**、**阴性结果守门人（NULL）**、**版本控制圣骑士（GIT）**、**经费召唤师（GRANT）**、**预印本冲锋手（PREPRINT）**。其中三个是隐藏人格，只有特定的答题模式才能解锁——包括那个大家最不想承认的：**"你最终变成了导师本人"（PI）**。

【配图 2｜21 种实验室生物图鉴】

- 用途：展示项目最有传播力的部分——人格本身。
- 构图建议：网格排列 21 个人格卡片（或精选 9–12 个），每张卡片含代码、中文名、一句标语；隐藏人格三张用半透明或"未解锁"锁形处理。
- 关键标注：BAYES／PIPET／R2D2／NULL／GIT／PI（隐藏）等。
- 图注文字：全部 21 种人格。名字与故事是原创虚构，原型向量是手工设计的——项目自己在方法页承认了这一点。
- 来源：建议直接截图应用内 `/atlas`（人格图鉴）页面。仓库内无现成截图文件。

### 二、真正难的不是好笑，是诚实

做一个搞笑测试不难。难的是当你决定"数学要是真的"之后，一连串工程问题就排着队来了。

第一个问题：**结果必须可复现**。同样的答案必须永远给出同样的人格，否则"确定性"就是一句空话。计分引擎里没有任何随机数；连按理说应该随机的自助重采样（bootstrap），种子也是由固定配置加答案哈希生成的——同一份答案，300 次重采样的结果每次都一模一样。平局怎么办？也写死了：先比加权距离，再比余弦相似度，最后按字母序。这个平局规则直接写在方法页上。

第二个问题：**内容本身需要质量控制**。36 道题、15 个维度、21 个人格、3 种语言（英文、简体、繁体），这些内容之间有大量交叉引用——题目的权重要指向真实存在的维度，隐藏规则要指向真实存在的题目和人格。项目为此写了一个内容校验脚本：每个 JSON 内容文件过一遍 Zod schema，再做跨文件引用完整性检查，最后检查三种语言的翻译键是否完全一致、有没有空字符串。这个校验跑在 CI 里，排在 lint 和单元测试前面。也就是说，**翻译漏一个键，整个部署直接失败**。

第三个问题：**重计算不能卡界面**。300 次重采样乘以每次完整的重新计分，放在主线程里会让页面卡顿，所以 bootstrap 被丢进了 Web Worker。PCA、k-means、层次聚类、熵，全部在浏览器里现算——没有后端，没有"上传答案到服务器分析"这回事。

这就是那个老生常谈的教训的又一个版本：一个"简单的小玩具"，认真做起来，四分之三的工作量花在用户永远不会注意到的地方。

### 三、整个项目里没有一行 AI

这可能是 2026 年最不合时宜的技术决策：LBTI 全程没有调用任何 LLM。

结果页那些"为你写的"解读文字，不是模型生成的，而是作者预先写好的文本模块，用一个确定性的模板引擎拼装——代码注释里明确写着 "never generative AI"。模板引擎甚至配了一个运行时守卫和对应测试，保证没有任何未填充的 `{占位符}` 会漏到界面上。

为什么？因为这个项目的全部立足点是**每一句话都可以追溯**。方法页把内容分成两栏：哪些是真实计算的（维度分数、距离、PCA、聚类、重采样、熵），哪些是编写的虚构（人格的名字与故事、原型向量、题目权重、所有解读）。一旦引入生成式模型，第二栏就会被污染——你无法为一段每次都不一样的生成文本负责，也无法把它放进 CI 里做完整性校验。

一个 AI 生成的报告，只有在读者能查到每句话的来源时才有价值。LBTI 的解法是极端的：干脆不生成，全部手写，然后用机器保证手写内容的完整性。这未必是所有项目的正确答案，但它把一个常被回避的问题摆上了台面——你的系统里，哪些文字是有人负责的？

- 用途：呈现项目最核心的设计哲学——把计算与虚构明确分账。
- 构图建议：左右两栏对照卡片。左栏"真实计算"：维度分数、加权距离、PCA、层次聚类、自助重采样、熵与分类间隔；右栏"编写的虚构"：人格与故事、原型向量、题目权重、所有解读文字。中间一条清晰的分界线。

### 四、当玩具需要一个数据库

LBTI 有一个可选功能：把你的结果匿名加入一张"群体图谱"——所有自愿参与者的点被实时投影到同一张 PCA 图上，点足够多时再跑 k-means 聚类。这需要一个共享数据库，项目用了 Supabase。

有意思的是权限设计。数据库迁移文件里的安全模型写得像一份威胁分析：匿名用户可以读、可以插入通过校验的行，但**不能更新、不能删除任何行**——没有对应的 RLS（Row Level Security）策略，默认拒绝。插入的数据只有派生的 15 维分数向量、人格代码和粗粒度上下文，数据库层面用 check 约束逐元素验证向量的每个值都在 0 到 100 之间。原始答案、姓名、邮箱、机构，从表结构上就不存在存放的地方。

换句话说：即使前端代码被人改了，数据库也只接受合法形状的匿名数据。校验不是只做在客户端的，这是很多"先上线再说"的项目最容易省掉的一层。

最近的开发分支还在加一个"意见信箱"（Letterbox）：用户可以提交新人格点子或新题目，写入两张只进不出的表——匿名用户只能插入，不能读取任何已有提交，审核状态字段对公众完全不可见。这个功能在本文写作时还未合入主分支，属于进行中的工作。

【配图 4｜隐私架构：什么数据在哪里】

- 用途：说明"数据不出浏览器"到底是怎么实现的。
- 构图建议：左侧一台笔记本电脑（浏览器内：36 条原始答案、localStorage、本地计算），右侧一个数据库图标（Supabase：仅 15 维派生向量 + 人格代码，insert-only），中间一条带闸门的窄通道，标注"可选、匿名、仅派生数据"。原始答案旁边画一个明确的"不出境"标记。
- 图注文字：原始答案永远留在浏览器里；共享图谱只接收派生分数，且数据库层面禁止更新与删除。

### 五、界限：一个玩具的自我修养

LBTI 在免责声明页写得很直白："所谓'分析'只是把简单算术演得很戏剧化。请不要据此做任何重要决定。"这不是套话，项目在几个容易被误读的地方都做了明确切割，这些切割本身值得一说：

**熵和"匹配强度"不是效度。** 结果页会给你一个匹配强度、分类间隔和归一化熵——它们衡量的是你的点在这个虚构原型空间里的位置是否清晰，而不是这个测试是否"准"。一个内部一致的系统照样可以整体上毫无意义，LBTI 只是罕见地把这句话自己说了出来。

【配图 5｜结果页：一份可以追问的"报告"】

- 用途：展示核心界面，让读者看到"匹配强度、熵、稳定性"在真实界面里的呈现方式。
- 构图建议：结果页截图，尽量在一屏内包含雷达图或维度条形图、匹配强度/分类间隔/熵指标区，以及自助重采样稳定性图；可在截图旁用细线标注三个关键读数的含义。
- 关键标注：15 维画像／匹配强度／归一化熵／300 次重采样稳定性。
- 图注文字：结果页的每个数字都有公式可查——它们衡量的是分类的清晰度，不是测试的"准确性"。
- 来源：需从运行中的应用 `/result` 路由截图（先完成一次测试）。仓库内无现成截图。

**"主题富集"没有 p 值。** 项目借用了生信里"富集分析"的说法，但方法页专门声明：这只是加权回答的相对集中程度，不是统计学的通路富集，不带错误发现率。贡献散点图借用了火山图的双轴语法，但坐标轴上没有 −log10(p)——因为根本没有假设检验，放那个数字就是撒谎。

**自助重采样测的是内部稳定性，不是信度。** 300 次重采样告诉你"换一批题目子集，你还会不会被分到同一个人格"，这是系统对扰动的敏感度，不是心理测量学意义上的重测信度。代码注释里原话是：这"不是临床信度或任何形式的科学验证"。

**隐藏人格不指控任何人。** 三个隐藏人格由明确的确定性规则触发（比如"组会隐身者"需要三个维度分数同时越过阈值），免责声明专门写了一句：它们是"对常见科研习惯饱含善意的漫画化"，从不指控学术不端。

这套切割功夫对任何做科研工具的人都有参考价值：**借用一门学科的视觉语法时，你有义务声明哪些含义没有被一起借来。** 画了个像火山图的东西，就得说清楚它不是火山图。

### 六、这跟正经做科研的你有什么关系

如果你是研究生或博后，LBTI 首先是个还不错的教学物件。它的"ML 实验室"页面把测试用到的每种方法——特征向量、缩放、距离、PCA、k-means、层次聚类、bootstrap、熵——都配了"学术表述"和"大白话"双语解释，外加可以拿去改的 Python 小模式（汇总重复、先标准化再投影、用小 ODE 模型模拟反馈系统）。你自己的答案会变成图上的一个点，跟着每一步变换走。用自己的数据学降维，比用鸢尾花数据集记得牢。

如果你写科研软件，这个仓库更像一份可抄的作业：

- **内容即数据。** 题目、权重、人格、规则全部是带版本号的 JSON，过 Zod schema，进 CI 校验。想改题目？改 JSON，校验不过就部署不了。这比把文案硬编码在组件里的项目多活五年。
- **把"诚实"做成页面。** 方法页不是 README 里一段没人看的免责声明，而是一个一级路由，和结果页平起平坐。
- **权限从数据库层做起。** insert-only 的 RLS 策略加上数据库层的形状校验，意味着信任模型不依赖前端代码的完好。
- **CI 里跑的是完整旅程。** GitHub Actions 工作流依次执行内容校验、lint、类型检查、单元测试、Playwright 端到端测试（在与 GitHub Pages 相同的子路径下跑桌面和移动端），全部通过才构建部署。对一个玩具来说这套关卡近乎奢侈，但这正是"个人脚本"和"可以放心给别人用的公开资源"之间的距离。

如果你做科普或科研社区产品，LBTI 演示了一种少见的定位：**不假装科学的科学传播。** 它的传播钩子（人格、分享卡、隐藏彩蛋）和它的教育内容（真实的数学、公开的方法）是同一套系统，前者把人骗进来，后者让人带点东西走。

### 七、局限

说完好话，说实话。

- **它终究是纯前端玩具的架构。** 所有计算在浏览器里完成，群体图谱的 PCA 和 k-means 也是客户端现算——参与者规模大了之后，每个访问者的浏览器都要拉全量记录来算投影，这个设计撑不了太大的量。
- **共享数据库没有防滥用机制。** 匿名插入虽然有严格的形状校验，但没有速率限制或去重（意见信箱有内容哈希去重，群体图谱没有）——一个写脚本的无聊人可以往图谱里灌合法形状的垃圾点。
- **意见信箱只进不出，审核纯靠人。** 提交进了数据库就沉底，模式里定义了七种审核状态，但整个审核流程目前没有任何工具界面，全靠维护者在 Supabase 后台手动看。
- **仓库声明 MIT 协议，但根目录还没有 LICENSE 文件**，只有 package.json 里的一行声明。严格来说，开源合规还差最后一步。
- **README 里的线上地址和仓库现名不一致**（仓库已改名，文档没跟上），端到端测试里也还硬编码着旧的子路径。这类"改了名字忘了改文档"的碎屑，恰好是项目自己嘲讽过的那种自动化暗伤。
- **没有监控。** 部署了就是部署了，Supabase 挂了、配额用完了，第一个发现的会是某个点开群体图谱看到报错的用户——虽然界面为这种情况准备了体面的降级文案。

这些都不是致命伤，而是一个诚实项目的诚实边界：它清楚自己是玩具，也把玩具该有的工程做到了明显超标的程度，但没有假装自己是基础设施。

### 结尾

LBTI 最值得记住的地方，不是 21 个好笑的人格名字，而是它处理"信任"的方式：每一个数字都能在方法页找到公式，每一段文字都能找到作者，每一条数据都能说清楚存在哪里、谁能动它。它甚至为"哪些东西是编的"专门做了一页。

我们看过太多反过来的系统——界面上挂满图表，报告里全是生成的漂亮句子，但你问一句"这个结论是从哪来的"，就没有人答得上来。一个工具值不值得用，不取决于它宣传了多少智能，而取决于用户能不能追问它、检查它、纠正它。LBTI 用一个玩笑把这件事做对了，这比很多严肃产品做得都好。

项目代码在 GitHub（sthsci/lumina-labtype），有想吐槽的人格或想加的题目，测试内建的意见信箱正等着你。
